在obsidian中打开，还可以看到wiki的图谱。使用以下方案：本地 AI 代理 + 文件夹监控>（自动化程度最高）                            
这非常契合当下最前沿的“活体维基（Live Wiki）”理念。你可以开发一个后台常驻的 macOS 小工具（或使用 Python                      
脚本打包），专门负责监听和编译知识。                                                                                         
核心逻辑：                                                                                                                   
文件夹监控：使用 watchdog 等库监控用户指定的 raw_papers 文件夹。一旦有新 PDF 拖>入，立刻触发处理流程。                        
本地/云端 AI 处理：在 Mac 上，你甚至可以集成 Ollama 来运行本地大模型（如 GLM-4-Flash、Llama 3                                
等），既保护隐私又能免费无限调用；当然也可以继续调用云端 API。                                                               
自动写入：AI 提取完摘要、实体并生成带双链的 Markdown 后，直接静默写入到 Obsidian 仓库的 wiki/ 目录下。
优势：全自动“无感”体验。用户只需要像平时一样把论文丢进文件夹，几分钟后 Obsidian 里就自动生成好了完美的学习笔记和关联图谱。
